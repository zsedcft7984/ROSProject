<주제>
아파트 단지 혹은 특정 주거 공간 내에서 라인으로 영역을 할당하고 특정장소에 마킹(QR)을 하여 
라인에 따라 주행하는 차량들이 영역내의 순찰을 통한 안전확보, 정해진 장소에 운송하는 기능을 제공함
<Frontend>
- 홈페이지 : 종합적인 정보 제공( 날씨, 공지사항) 및 로그인
- 일반사용자와 관리자의 아이디를 나누어 서로 접근할 수 있는 정보가 다름
- 관리자 페이지 : 택배의 배달 정보 제어, 순찰정보, 공지사항 관리 등의 기능 등
- 회원 기능 
<Backend>
- 관리자와 일반사용자 아이디 분리
- 날씨 API 받아와서 사용
- 회원가입시 전화번호, 이름, 동호수, 아이디, 비밀번호
- 공지사항에서 개시물별 조회수 확인기능 
- 개시물에 첨부파일있으면 다운기능 (상세내역에서도 기능)  - 보류

- 개인의 택배 내역 및 배송상태 조회기능
- 관리자 페이지에서 공지사항, 입주민 등록 및 수정, 삭제 기능
- 회원 탈퇴기능
- 택배 차량 배송 정보 및 동작 기능
- 순찰 차량 수동제어  
- 순찰 및 거수자 정보 관리리

<Machine Learning>
- 입주민 데이터 학습 모델 생성
- 흉기 데이터 학습 모델 생성
- yolo 모델을 webcam에서 인식
- yolo 모델을 ros 차량에서 제공한 영상에 적용
- 인식이 되면 사진을 저장( 라벨명 - 숫자 - 날짜)
- 사진 저장시 ros로 특정 신호 전송하도록(부저동작)
- 저장된 사진의 경우 데이터 베이스를 통해 왭페이지에 등록되도록함
- 또한 특정 사진의 경우 왭페이지에 log가 남도록함함
<ROS>
(1) 순찰 로봇
- 기본적으로 정지상태 유지
- 웹에서 라인트레이싱 on/off 버튼으로 제어가능
- 자동 상태일때는 라인트레이을 바탕으로 이동
- 웹캠을 통해 웹에서 영상 확인가능
- yolo를 통해 학습된 모델의 이미지가 보이면 부져 울림
- 수동모드일 경우
- 방향조작가능
- 속도 제어가능
- 부저 on/off 가능
(2) 운송 로봇
- 웹에서 배달해야 하는 위치를 체크
- 버튼 클릭시 배송상태 변경(배송중으로) 및 ROS로 신호
- 파란색 라인에 따라 주행 
- 주행 중 전송받은 데이터(문자열 형태)가 포함된 QR 감지시 정지
- 정지시 해당 QR의 정보가 웹으로 전송
- 전송받은 데이터를 모두 완료하면 home QR이 보일떄 까지 주행
- Home 인식하면 운행 종료
- 배송상태를 배송완료로 바꿈꿈
